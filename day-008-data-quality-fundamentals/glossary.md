# Day 008 Glossary

| Term | Simple Meaning | Beginner Example |
| --- | --- | --- |
| Data Quality | How trustworthy data is for its use. | Sales data matches real orders. |
| Accuracy | Data correctly represents reality. | Order amount matches payment. |
| Completeness | Required data is present. | Every order has an order ID. |
| Consistency | Data is represented the same way. | Country is always `UK`. |
| Validity | Data follows expected rules. | Email contains `@`. |
| Uniqueness | Records that should be unique are not repeated. | One transaction ID appears once. |
| Timeliness | Data is available when needed. | Report is ready before work starts. |
| Freshness | Data was updated recently enough. | Dashboard refreshed today. |
| Integrity | Relationships between datasets are correct. | Order customer exists in customer table. |
| Reliability | Data can be trusted repeatedly. | Same metric is stable over time. |
| Missing Value | Required value is absent. | Blank customer ID. |
| Null | A marker for missing or unknown value. | Country is null. |
| Duplicate Record | Same record appears more than once. | Same order loaded twice. |
| Invalid Format | Value does not match expected pattern. | Invalid date format. |
| Wrong Data Type | Value is stored as wrong kind of data. | Amount stored as text. |
| Inconsistent Value | Same meaning written differently. | `Paid`, `paid`, `P`. |
| Out-of-Range Value | Value is outside expected limits. | Discount is 150%. |
| Stale Data | Old data not refreshed as expected. | Dashboard shows last week. |
| Broken Relationship | Related records do not match. | Payment has no matching order. |
| Validation Check | Rule to confirm data looks correct. | Check order ID is not null. |
| Row Count Check | Check number of records. | Source has 1,000 rows. |
| Null Check | Check missing values. | Order amount is not null. |
| Duplicate Check | Check repeated records. | Transaction ID is unique. |
| Data Type Check | Check value type. | Amount is numeric. |
| Range Check | Check value is within limits. | Rating is between 1 and 5. |
| Accepted Values Check | Check value is from allowed list. | Status is Paid or Failed. |
| Source-to-Target Check | Compare source and loaded data. | Source and target totals match. |
| Freshness Check | Check data is updated on time. | Dashboard refreshed by 8 AM. |
| Anomaly Check | Check unusual patterns. | Sales suddenly drops to zero. |
| Business Rule | Rule based on business meaning. | Refund cannot exceed purchase. |
| Technical Check | Rule based on structure or format. | File has expected columns. |
| Quarantine | Separate bad records for review. | Invalid rows moved aside. |
| Alert | Notification about an issue. | Pipeline failure message. |
| Retry | Try a failed process again. | Rerun after file arrives. |
| Reprocessing | Process data again. | Rebuild yesterday's sales table. |
| Data Trust | Confidence that data is reliable. | Analysts believe dashboard numbers. |
| Data Issue | Any problem with data quality. | Missing values or duplicates. |
