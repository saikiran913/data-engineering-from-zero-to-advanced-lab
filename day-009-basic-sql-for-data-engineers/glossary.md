# Day 009 Glossary

| Term | Simple Meaning | Beginner Example |
| --- | --- | --- |
| SQL | Structured Query Language used to work with databases. | Read customers from a table. |
| Query | A SQL instruction or question. | Show failed payments. |
| Database | System that stores tables. | Customer database. |
| Data Warehouse | System for reporting and analytics tables. | Sales warehouse. |
| Table | Data arranged in rows and columns. | `orders` table. |
| Row | One complete record. | One order. |
| Record | One item of data. | One customer record. |
| Column | One attribute in a table. | `country`. |
| Field | A data attribute or value. | `order_amount`. |
| SELECT | Chooses columns to return. | `SELECT customer_name` |
| FROM | Chooses table to read. | `FROM customers` |
| WHERE | Filters rows. | `WHERE country = 'UK'` |
| ORDER BY | Sorts results. | `ORDER BY order_amount DESC` |
| ASC | Ascending sort. | Oldest to newest. |
| DESC | Descending sort. | Highest to lowest. |
| LIMIT | Restricts rows returned. | `LIMIT 10` |
| Filter | Keep only matching rows. | Only active customers. |
| Condition | Rule used in a filter. | `order_amount > 500` |
| Operator | Symbol used for comparison. | `=`, `>`, `<` |
| Text Value | Word or text data. | `'Active'` |
| Numeric Value | Number data. | `500` |
| Date Value | Date data. | `'2026-06-01'` |
| SELECT * | Select all columns. | Preview all customer fields. |
| Clause | Part of a SQL query. | `WHERE`, `ORDER BY` |
| Semicolon | Ends a SQL statement in many tools. | `;` |
| Result Set | Output returned by a query. | Filtered rows. |
| Syntax | Rules of writing SQL. | Correct keyword order. |
| Error Message | Message when SQL has a problem. | Unknown column name. |
| Data Preview | Small look at table data. | First 10 rows. |
| Data Investigation | Using data to understand an issue. | Review failed payments. |
