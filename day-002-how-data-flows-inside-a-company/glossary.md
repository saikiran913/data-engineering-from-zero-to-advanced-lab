# Day 002 Glossary

| Term | Simple Meaning | Beginner Example |
| --- | --- | --- |
| Data Flow | The journey data takes from where it is created to where it is used. | An order moves from a website to a sales dashboard. |
| Source System | The system where data starts or is first stored. | A website, mobile app, payment system, or database. |
| Application | Software that users interact with to perform actions. | A food delivery app where a customer places an order. |
| Operational Database | A database used to support daily business activity. | An orders database used by an online store. |
| Data Ingestion | Collecting data from a source and bringing it into a data platform. | Loading order data from a database into storage every hour. |
| Data Pipeline | A process that moves and prepares data through different steps. | A pipeline that moves raw orders, cleans them, and creates a sales table. |
| Raw Data | Data stored close to its original form. | An order record exactly as received from the website. |
| Raw Layer | The storage layer where original data is kept. | Raw files copied from an application database. |
| Clean Layer | The layer where data is corrected and standardized. | Orders with fixed dates, valid amounts, and no duplicates. |
| Business Layer | The layer where data is ready for dashboards and decisions. | A daily sales summary table used by the sales team. |
| Data Cleaning | Fixing messy data so it becomes more usable. | Changing `USER@EMAIL.COM` to `user@email.com`. |
| Data Transformation | Changing data into a useful business-ready structure. | Calculating total sales by day. |
| Data Warehouse | A system that stores clean, structured data for analytics. | A warehouse table used by a sales dashboard. |
| Dashboard | A visual report that shows business performance. | A Power BI report showing revenue and order count. |
| Business Logic | Rules that define how business numbers should be calculated. | Revenue equals successful order amount minus refunds. |
| Data Quality Issue | A problem that makes data less accurate or less trustworthy. | Missing customer ID or duplicate order records. |
| Pipeline Failure | When a data pipeline stops or does not complete successfully. | A daily sales table is not updated because the pipeline failed. |
| Historical Data | Past data kept for analysis, audit, or comparison. | Last year's sales records. |
| Reporting System | A system used to create reports and dashboards. | Power BI, Tableau, or Looker Studio. |
| Analytics | Using data to understand patterns, performance, and decisions. | Studying which products sell best each month. |
