## Answer Key

### Section 1

1. Structured data is data organized in rows and columns.
2. Semi-structured data has some organization but does not always fit neatly into rows and columns.
3. Unstructured data has no fixed table-like structure.
4. Structured data is easier because it looks like a clear table.
5. JSON is semi-structured because it has keys and values, but can also have nested sections.
6. Examples include images, videos, audio recordings, PDFs, emails, and documents.
7. Data engineers care because data type affects storage, cleaning, transformation, and analysis.
8. Parsing means reading data and breaking it into useful parts.
9. SQL works well with rows and columns.
10. Unstructured data may need OCR, text processing, AI, metadata extraction, or other processing before analysis.
