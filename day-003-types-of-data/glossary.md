# Day 003 Glossary

| Term | Simple Meaning | Beginner Example |
| --- | --- | --- |
| Structured Data | Data organized in rows and columns. | A customer table with customer ID, name, and country. |
| Semi-Structured Data | Data with some organization, but not always table-like. | A JSON API response with nested order details. |
| Unstructured Data | Data without a fixed table-like structure. | A video file, email message, or product image. |
| Row | One record in a table. | One customer's details in a customer table. |
| Column | One field or attribute in a table. | `customer_name` or `order_amount`. |
| Table | Data arranged in rows and columns. | An orders table in a database. |
| Schema | The defined structure of data, such as table names and columns. | An orders table must have `order_id`, `amount`, and `order_date`. |
| JSON | A common semi-structured format used by APIs and applications. | `{ "order_id": 101, "amount": 500 }` |
| XML | A semi-structured format that uses tags to describe data. | `<order><amount>500</amount></order>` |
| Log File | A file that records events or activities from a system. | A server log showing page visits and errors. |
| API Response | Data returned by an API after a request. | A payment API returns payment status and transaction ID. |
| Event Data | Data created when something happens in an app or system. | A user clicks a product or watches a video. |
| Metadata | Data that describes other data. | Image file size, upload date, and file type. |
| Object Storage | Storage commonly used for files such as images, videos, logs, and documents. | A cloud storage bucket holding product images. |
| Data Lake | A storage area for large amounts of raw or processed data in many formats. | A place storing CSV files, JSON logs, images, and PDFs. |
| Database | A system used to store and manage data. | A database storing customer and order tables. |
| Data Warehouse | A system used to store structured data for analytics and reporting. | A warehouse table used by a sales dashboard. |
| Parsing | Reading data and breaking it into useful parts. | Extracting `customer_id` and `amount` from JSON. |
| File Format | The way information is stored inside a file. | CSV, JSON, XML, PDF, PNG, or MP4. |
| Text Data | Data made of written words or characters. | Customer reviews, emails, or chat messages. |
